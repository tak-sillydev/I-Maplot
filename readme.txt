xml -> Pref -> City -> Code の上４桁が「行政区域コード」。
https://nlftp.mlit.go.jp/ksj/gml/datalist/KsjTmplt-P34.html
このサイトから行政区域コードとその役場の緯度経度が入ったシェープファイルを入手可能なので、それをもとに描画

震央座標は（国内で発生したものなら）日本測地系。
細分区域図は世界測地系で作られているので変換する必要あり？

変更予定？
　プロセス名の変更。当該システムを見つけやすくする？
　震度速報か震源震度情報かの判断を電文番号（VXSE51/VXSE53）で行う？
　試験電文の誤発報防止措置
　例外発生箇所の行番号の記述
　インターネット接続が切れたときに例外を発生、ロギングしようとするもメールによるログ送信ができず二重例外で無言で落ちてしまうバグの修正
	（コンソール画面にはトレースバックが書かれている）
　alive.py に送出するデータにデータの最新取得時刻（と最新地震発生時刻）を載せる

変更履歴：
v1.0.0
  - JMAEQ I-Maplot 公開。震度地図を描画する。
v1.0.1
  - report.py における震度描画基準決定プロセスの変更。
v1.0.2
  - 震源の深さ／マグニチュードが不明な場合の処理に対応。
v1.1.0
  - config_.json 追加。雛形なので実際に使用するときはデータを入力した上でファイル名からアンダーバーを取ること。
  - 生存確認機能（alive.py）の実装。
  - config.json の構成変更。生存確認（alive.py）、システム終了（exit.py）のための情報を追加した。
  - 連続で地震情報の取得に失敗した場合、指定回数おきにERRORレベルのログを吐く。
  - 地震情報がない場合のログレベルをINFOからDEBUGに変更。
  - exit.py 実行時に、コマンドライン引数を用いた独自の終了メッセージが送信可能になった。
      あわせて、送受信するメッセージの構造を変更。
  - If-Modified-Since HTTPヘッダを追加。情報更新時にのみデータがダウンロードされるようになった。
v1.1.1
  - ロギングシステムの改良。アプリからのログのみ吐き出すようになった。
v1.1.2
  - 遠隔環境からaliveコマンドとexitコマンドを送信できるようになった。
  - aliveコマンドの応答に最新データ取得時刻と最新の地震情報を乗せるように変更。
